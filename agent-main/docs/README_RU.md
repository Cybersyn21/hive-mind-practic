# Каталог docs/ — Документация

## Описание

Каталог `docs/` содержит дополнительную документацию проекта, включая case studies (примеры использования) и технические исследования.

## Структура

```
docs/
└── case-studies/                    # Практические примеры
    ├── installation-error/          # Ошибка установки
    ├── playwright-mcp-support/      # Поддержка Playwright MCP
    └── system-message-options/      # Опции системных сообщений
```

## Case Studies (Практические примеры)

### installation-error/
**Тема**: Ошибка установки агента

**Файлы**:
- `case-study.md` — Описание проблемы и решения
- `issue-5.json` — Данные связанного issue

**Содержание**: Документация по диагностике и решению проблем при установке пакета.

### playwright-mcp-support/
**Тема**: Поддержка Playwright через MCP

**Файлы**:
- `case-study.md` — Исследование интеграции
- `issue-16.json` — Данные связанного issue
- `playwright-mcp-research.json` — Результаты исследования

**Содержание**: Анализ возможности интеграции Playwright для автоматизации браузера через Model Context Protocol.

### system-message-options/
**Тема**: Опции системных сообщений

**Файлы**:
- `case-study.md` — Документация опций
- `findings.json` — Результаты исследования

**Содержание**: Исследование различных подходов к настройке системных промптов и их влияния на поведение агента.

## Формат Case Study

Каждый case study содержит:

1. **Описание проблемы** — Что нужно было решить
2. **Исследование** — Какие подходы рассматривались
3. **Решение** — Что было выбрано и почему
4. **Результаты** — Итоги и выводы
5. **Код/Конфигурация** — Примеры реализации

## Добавление нового Case Study

1. Создайте директорию с понятным именем
2. Добавьте `case-study.md` с описанием
3. Приложите связанные данные (JSON, логи)
4. Обновите этот README
