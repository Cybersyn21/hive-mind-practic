# Каталог experiments/ — Эксперименты

## Описание

Каталог `experiments/` содержит экспериментальные скрипты и исследования для тестирования новых функций и подходов.

## Структура

```
experiments/
└── solve-dockerize/          # Эксперименты с Docker изоляцией solve
```

## solve-dockerize/

**Тема**: Запуск solve в изолированном Docker контейнере.

**Цель**: Исследование возможности полной изоляции процесса решения issues.

**Преимущества изоляции**:
- Безопасность — AI не может повредить хост
- Воспроизводимость — идентичное окружение
- Масштабируемость — легко запускать множество экземпляров

**Ограничения**:
- Усложнение архитектуры
- Необходимость передачи авторизации
- Overhead на создание контейнеров

## Создание эксперимента

### 1. Создайте директорию
```bash
mkdir experiments/my-experiment
```

### 2. Добавьте файлы
```bash
# Основной скрипт
touch experiments/my-experiment/run.mjs

# Документация
touch experiments/my-experiment/README.md
```

### 3. Документируйте
```markdown
# Название эксперимента

## Цель
Что исследуем и зачем.

## Гипотеза
Что ожидаем получить.

## Методология
Как проводим эксперимент.

## Результаты
Что получили.

## Выводы
Какие решения приняты.
```

### 4. Запустите
```bash
node experiments/my-experiment/run.mjs
```

## Миграция в production

Если эксперимент успешен:

1. Перенесите код в `src/`
2. Добавьте тесты в `tests/`
3. Обновите документацию в `docs/`
4. Удалите или архивируйте эксперимент

## Примечания

- Эксперименты не обязаны быть стабильными
- Можно ломать совместимость
- Документируйте результаты
- Удаляйте неудачные эксперименты
